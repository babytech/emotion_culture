# 第五阶段 M1 真机冒烟验证清单

## 文档目的

本文件用于在 Phase5 正式进入 `M2` 前，先对当前已完成的 `M1` 起步改动做一次最小真机验证。

当前只覆盖以下 4 类能力：

- `MINI-511` 首次进入授权页与首页前置拦截
- `BE-511` 启动信息接口与微信身份自检
- `MINI-513` 自拍前摄像头权限前置校验
- `MINI-514` 录音中提交自动收口

建议使用：

- 1 台 iPhone 真机
- 1 台 Android 真机
- 体验版小程序

## 测试前准备

### 环境要求

- 小程序已更新到包含当前 Phase5 M1 代码的体验版
- 云托管后端已部署包含 `/api/bootstrap` 的版本
- 测试微信号能正常访问当前体验版

### 开始前操作

- 删除小程序本地缓存，确保能模拟“首次进入”
- 确认此前未残留授权门本地状态
- 准备一段可录音的安静环境

## 冒烟范围

本轮只验证以下问题是否已经被修正：

- 打开小程序后是否先进入授权门
- 授权门完成后是否能进入主 tab
- 自拍是否先显式要求摄像头权限
- 录音中直接提交时，是否会先自动结束录音再继续分析
- 权限拒绝或停止录音失败时，是否会给出明确提示

## 用例清单

### CASE-511-01 首次进入必须先落授权门

- 前置条件：清除小程序缓存，确保本地不存在授权门完成状态
- 操作步骤：
- 打开体验版小程序
- 观察首屏落点
- 预期结果：
- 不应直接进入首页、分析页或其它 tab
- 应先进入“微信身份确认 / 授权门”页面

### CASE-511-02 同意进入后跳转主 tab

- 前置条件：位于授权门页面，后端 `/api/bootstrap` 可正常返回身份信息
- 操作步骤：
- 点击“同意并使用微信身份进入首页”
- 观察跳转结果
- 预期结果：
- 页面可正常离开授权门
- 默认进入首页
- 后续切换 `首页 / 记录 / 分析 / 收藏 / 我的` 不再被重复拦截

### CASE-511-03 已完成授权后再次打开不重复拦截

- 前置条件：`CASE-511-02` 已通过
- 操作步骤：
- 关闭小程序，再次打开
- 或切后台后重新进入
- 预期结果：
- 不再重复停留在授权门
- 应直接进入上次允许进入的主路径

### CASE-511-04 bootstrap 接口异常时可见失败态

- 前置条件：人为制造后端不可用或临时断网
- 操作步骤：
- 打开授权门
- 观察状态区域与按钮表现
- 预期结果：
- 页面不白屏
- 会显示“身份初始化失败，请稍后重试”或等价提示
- 可点击“重新获取身份”

### CASE-513-01 自拍前先显式要求摄像头权限

- 前置条件：系统尚未授权摄像头给该小程序
- 操作步骤：
- 进入分析页
- 点击“前置自拍”
- 预期结果：
- 不会直接进入拍摄
- 应先出现权限说明或授权确认流程
- 同意后才进入拍摄

### CASE-513-02 拒绝摄像头权限后直接返回

- 前置条件：`CASE-513-01` 进行中
- 操作步骤：
- 在摄像头授权流程中选择拒绝
- 观察页面行为
- 预期结果：
- 不应继续打开拍摄界面
- 应留在分析页
- 应给出明确提示，或引导用户去设置中开启

### CASE-513-03 摄像头权限已关闭时可从设置恢复

- 前置条件：系统设置中已关闭该小程序摄像头权限
- 操作步骤：
- 点击“前置自拍”
- 选择“去设置”
- 在设置中开启摄像头权限
- 返回小程序后再次点击“前置自拍”
- 预期结果：
- 设置恢复路径可闭环
- 恢复后可正常进入拍摄

### CASE-514-01 录音中直接提交会自动收口

- 前置条件：进入分析页
- 操作步骤：
- 点击“开始录音”
- 在仍处于录音中时直接点击“开始分析”
- 预期结果：
- 不会静默丢失录音
- 系统会先尝试自动结束录音
- 停止成功后继续进入提交/上传/分析流程

### CASE-514-02 自动结束录音失败时会阻断提交

- 前置条件：模拟停止录音失败，或在录音状态异常时触发提交
- 操作步骤：
- 录音过程中直接点击“开始分析”
- 观察错误提示
- 预期结果：
- 提交被阻断
- 用户能看到“请先结束录音再提交”或等价提示
- 不会进入一个缺失语音的结果页

### CASE-514-03 录音结束后结果页可带出语音链路

- 前置条件：正常录音并结束
- 操作步骤：
- 点击“开始分析”
- 等待进入结果页
- 如当前链路支持，继续尝试发送邮件
- 预期结果：
- 结果页不应出现“我录了但系统没带上”的明显错觉
- 若当前语音识别成功，应能看到语音相关结果
- 邮件链路不应因这次录音治理改动而回退

## 建议记录格式

每条用例建议记录以下字段：

- 设备：`iPhone / Android`
- 系统版本
- 微信版本
- 小程序版本 / 体验版时间
- 结果：`PASS / FAIL`
- 现象截图或录屏
- 补充说明

## 通过标准

只有当以下 4 条同时满足，才建议继续进入 `M2` 主开发：

- 首次进入不再直接落首页
- 授权门完成后可稳定进入主 tab
- 自拍权限路径清晰，拒绝后可恢复
- 录音中直接提交不再静默丢失语音

## 测后建议

- 若冒烟全部通过：继续推进 `MINI-521` 与 `MINI-523`
- 若授权门或权限链路失败：优先修复 `M1`，不要立即并行推进 `M2`
- 若只是文案或视觉细节问题：可记录为次级修正，不阻塞 `M2`
