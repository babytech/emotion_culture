# 第五阶段 M1 真机冒烟验证清单

## 文档目的

本文件用于在 Phase5 正式进入 `M2` 前，先对当前已完成的 `M1` 起步改动做一次最小真机验证。

当前只覆盖以下 5 类能力：

- `MINI-511` 首次进入授权页与首页前置拦截
- `BE-511` 启动信息接口与微信身份自检
- `MINI-512` 隐私授权后使用当前微信身份进入
- `MINI-513` 自拍前摄像头权限前置校验
- `MINI-514` 录音中提交自动收口

建议使用：

- 1 台 iPhone 真机
- 1 台 Android 真机
- 体验版小程序

## 测试前准备

### 环境要求

- 小程序已更新到包含当前 Phase5 M1 代码的体验版
- 云托管后端已部署包含 `/api/bootstrap` 的版本
- 测试微信号能正常访问当前体验版
- 当前版本登录主键策略为：`unionid` 优先，`openid` 兜底
- 历史记录、会员、积分均绑定到当前微信身份

### 开始前操作

- 删除小程序本地缓存，确保能模拟“首次进入”
- 确认此前未残留授权门本地状态
- 准备一段可录音的安静环境

## 冒烟范围

本轮只验证以下问题是否已经被修正：

- 打开小程序后是否先进入授权门
- 授权门是否按“同意 -> 使用当前微信身份进入 -> 已登录”顺序完成
- 授权门完成后是否默认进入首页
- 自拍是否先显式要求摄像头权限
- 录音中直接提交时，是否会先自动结束录音再继续分析
- 权限拒绝或停止录音失败时，是否会给出明确提示

## 用例清单

### CASE-511-01 首次进入必须先落授权门

- 前置条件：清除小程序缓存，确保本地不存在授权门完成状态
- 操作步骤：
- 打开体验版小程序
- 观察首屏落点
- 预期结果：
- 不应直接进入首页、分析页或其它 tab
- 应先进入“微信身份确认 / 授权门”页面

### CASE-511-02 点击同意后进入待进入状态

- 前置条件：位于授权门页面
- 操作步骤：
- 点击“同意”
- 观察授权门页面状态
- 预期结果：
- 页面不应直接离开授权门
- 页面状态从“未登录”切换为“待进入”或等价表达
- “使用当前微信身份进入”按钮进入可点击状态

### CASE-511-03 点击使用当前微信身份进入后进入首页

- 前置条件：`CASE-511-02` 已通过，后端 `/api/bootstrap` 可正常返回当前微信身份
- 操作步骤：
- 点击“使用当前微信身份进入”
- 观察授权完成后的跳转结果
- 预期结果：
- 页面状态短暂显示“已登录”或等价成功态
- 页面可正常离开授权门
- 默认进入首页
- 后续切换 `首页 / 记录 / 分析 / 收藏 / 我的` 不再被重复拦截

### CASE-511-04 已完成授权后再次打开不重复拦截

- 前置条件：`CASE-511-03` 已通过
- 操作步骤：
- 关闭小程序，再次打开
- 或切后台后重新进入
- 预期结果：
- 不再重复停留在授权门
- 应直接进入首页或上次主 tab，不应再次卡在认证页

### CASE-511-05 未同意时直接进入会被拦住

- 前置条件：位于授权门页面，尚未点击“同意”
- 操作步骤：
- 直接点击“使用当前微信身份进入”
- 观察页面表现
- 预期结果：
- 页面不应误判为已登录
- 应留在授权门页面
- 应给出“请先点击同意”或等价提示

### CASE-511-06 bootstrap 或 auth 接口异常时可见失败态

- 前置条件：人为制造后端不可用或临时断网
- 操作步骤：
- 打开授权门
- 完成“同意”后点击“使用当前微信身份进入”
- 观察状态区域与提示表现
- 预期结果：
- 页面不白屏
- 微信身份初始化失败时可见明确错误提示
- 页面仍停留在授权门，可继续重试

### CASE-513-01 自拍前先显式要求摄像头权限

- 前置条件：系统尚未授权摄像头给该小程序
- 操作步骤：
- 进入分析页
- 点击“前置自拍”
- 预期结果：
- 不会直接进入拍摄
- 应先出现权限说明或授权确认流程
- 同意后才进入拍摄

### CASE-513-02 拒绝摄像头权限后直接返回

- 前置条件：`CASE-513-01` 进行中
- 操作步骤：
- 在摄像头授权流程中选择拒绝
- 观察页面行为
- 预期结果：
- 不应继续打开拍摄界面
- 应留在分析页
- 应给出明确提示，或引导用户去设置中开启

### CASE-513-03 摄像头权限已关闭时可从设置恢复

- 前置条件：系统设置中已关闭该小程序摄像头权限
- 操作步骤：
- 点击“前置自拍”
- 选择“去设置”
- 在设置中开启摄像头权限
- 返回小程序后再次点击“前置自拍”
- 预期结果：
- 设置恢复路径可闭环
- 恢复后可正常进入拍摄

### CASE-514-01 录音中直接提交会自动收口

- 前置条件：进入分析页
- 操作步骤：
- 点击“开始录音”
- 在仍处于录音中时直接点击“开始分析”
- 预期结果：
- 不会静默丢失录音
- 系统会先尝试自动结束录音
- 停止成功后继续进入提交/上传/分析流程

### CASE-514-02 自动结束录音失败时会阻断提交

- 前置条件：模拟停止录音失败，或在录音状态异常时触发提交
- 操作步骤：
- 录音过程中直接点击“开始分析”
- 观察错误提示
- 预期结果：
- 提交被阻断
- 用户能看到“请先结束录音再提交”或等价提示
- 不会进入一个缺失语音的结果页

### CASE-514-03 录音结束后结果页可带出语音链路

- 前置条件：正常录音并结束
- 操作步骤：
- 点击“开始分析”
- 等待进入结果页
- 如当前链路支持，继续尝试发送邮件
- 预期结果：
- 结果页不应出现“我录了但系统没带上”的明显错觉
- 若当前语音识别成功，应能看到语音相关结果
- 邮件链路不应因这次录音治理改动而回退

## 建议记录格式

每条用例建议记录以下字段：

- 设备：`iPhone / Android`
- 系统版本
- 微信版本
- 小程序版本 / 体验版时间
- 结果：`PASS / FAIL`
- 现象截图或录屏
- 补充说明

## 通过标准

只有当以下 4 条同时满足，才建议继续进入 `M2` 主开发：

- 首次进入不再直接落首页
- 认证流程符合“同意 -> 使用当前微信身份进入 -> 已登录 -> 首页”
- 自拍权限路径清晰，拒绝后可恢复
- 录音中直接提交不再静默丢失语音

## 审核通过后第一轮真机复测清单

本节用于在“用户隐私保护指引”后台状态从 `审核中` 变为 `已更新 / 已生效` 后，做第一轮专项复测。

建议执行顺序：

1. 确认后台隐私协议状态已不是 `审核中`
2. 重新上传最新体验版
3. 真机删除小程序缓存
4. 重新扫码进入并按下述用例回归

### 前置确认

- 微信公众平台中“用户隐私保护指引”状态已更新
- 最新体验版已包含当前“微信身份直入”版本代码
- 真机已删除小程序缓存，避免命中旧授权状态
- 云托管后端仍是包含 `/api/bootstrap` 的最新版本

### CASE-512-R1 首次进入落登录弹层

- 操作步骤：
- 删除缓存后重新打开体验版
- 观察首屏样式与交互
- 预期结果：
- 不直接进入首页
- 页面为“背景页被遮罩 + 底部登录弹层”样式
- 可看到“同意”与“使用当前微信身份进入”两个主操作

### CASE-512-R2 点击同意后完成隐私授权

- 操作步骤：
- 点击“同意”
- 观察微信隐私授权链路与页面状态变化
- 预期结果：
- 若微信拉起隐私授权确认，应能正常完成
- 完成后“同意”按钮应变为已完成态
- 页面不应出现“没有地方点同意，但又提示先点同意”的矛盾状态

### CASE-512-R3 点击“使用当前微信身份进入”后完成身份初始化

- 前置条件：`CASE-512-R2` 已通过
- 操作步骤：
- 点击“使用当前微信身份进入”
- 观察页面与跳转结果
- 预期结果：
- 不依赖手机号原生面板
- 应使用当前微信身份完成初始化
- 不应再出现“当前小程序暂不具备手机号能力”的错误提示

### CASE-512-R4 完成微信身份初始化后进入首页

- 前置条件：`CASE-512-R3` 已通过
- 操作步骤：
- 等待页面跳转
- 预期结果：
- 登录页短暂显示成功态后离开
- 默认进入首页
- 后续切换 `首页 / 记录 / 分析 / 收藏 / 我的` 不再重复拦截

### CASE-512-R5 再次打开不重复卡登录页

- 前置条件：`CASE-512-R4` 已通过
- 操作步骤：
- 关闭小程序
- 再次打开或切后台再进入
- 预期结果：
- 不应再次停留在登录弹层
- 应直接进入首页或上次主 tab

### CASE-512-R6 当前微信身份进入失败时仍留在登录页

- 前置条件：已完成“同意”步骤，且人为制造网络或环境异常
- 操作步骤：
- 点击“使用当前微信身份进入”
- 预期结果：
- 不应误判为已登录
- 仍停留在登录页
- 应提示“微信身份初始化失败，请稍后重试”或等价表达

### CASE-512-R7 后台配置错误时提示可判读

- 前置条件：使用一台可查看开发者工具日志的设备或联调环境
- 操作步骤：
- 重复执行“同意 -> 使用当前微信身份进入”
- 观察页面提示与控制台日志
- 预期结果：
- 若仍失败，提示应能区分：
- `请先点击同意`
- `当前微信身份初始化失败，请在微信内重新打开小程序`
- `微信身份初始化失败，请稍后重试`
- 开发者工具控制台可看到 `/api/bootstrap` 相关请求结果

### CASE-512-R8 已登录后分析主链路不受回归影响

- 前置条件：`CASE-512-R4` 已通过
- 操作步骤：
- 进入分析页
- 测试自拍
- 测试录音
- 提交分析并进入结果页
- 预期结果：
- 登录修复不应影响自拍、录音与分析提交流程
- 结果页可正常打开
- 返回首页后不应再次触发登录拦截

## 测后建议

- 若冒烟全部通过：继续推进 `MINI-521` 与 `MINI-523`
- 若授权门或权限链路失败：优先修复 `M1`，不要立即并行推进 `M2`
- 若只是文案或视觉细节问题：可记录为次级修正，不阻塞 `M2`
