# 第三阶段开发计划（动态图能力稳定化，必须完成）

## 文档目的

本文件用于明确第三阶段的核心目标：将诗人/国潮图片能力从“静态资源可用”升级为“可接入腾讯云或第三方动态图服务且在真机稳定可用”。

第三阶段以“稳定链路优先”而非“玩法数量优先”为原则，配套任务拆解见：

- [phase3-task-breakdown.md](/Users/babytech/github/emotion_culture/docs/phase3-task-breakdown.md)
- [phase3-implementation-checklist.md](/Users/babytech/github/emotion_culture/docs/phase3-implementation-checklist.md)

## 第三阶段目标（Must）

第三阶段结束时，必须达成以下结果：

- 小程序不再直接依赖第三方图片域名加载动态图
- 动态图统一走后端中转与自有存储域名（COS/云存储）输出
- 小程序结果页支持“云端动态图失败 -> 本地静态图兜底”
- 邮件与结果页图片来源口径一致，不再出现“邮件有图、页面无图”
- 形成可替换供应商的适配层（腾讯云/第三方如 liblib）
- 动态生图只能由用户主动点击触发，不允许后台自动触发
- 动态生图前必须明确告知“将使用用户自拍作为原图并发送到第三方服务”并获得同意
- 用户拒绝授权时立即取消生图，不得继续调用第三方
- 动态生图必须支持用户指定风格：`科技类` 或 `国潮类`
- 同一用户每自然周最多生图 `1` 次，超限必须拒绝并给出明确提示
- 每次生图均扣减用户积分（失败需可回滚），上线前需具备积分能力
- 用户识别仅使用微信身份（`openid/unionid`），不引入独立账号密码体系

## 核心方案（固定）

采用以下固定链路，不在本阶段再讨论替代主链：

1. 用户在结果页主动点击“生成动态图”并选择风格（科技/国潮）
2. 前端展示授权说明并征得同意（使用自拍原图 + 第三方生成）
3. 后端基于微信身份校验周限额与积分余额
4. 校验通过后创建异步任务，调用腾讯云或第三方服务生成图片
5. 后端下载生成结果并上传到自有存储
6. 后端仅返回自有域名 URL 或 file_id
7. 小程序仅加载自有域名图片，失败时回退本地静态图

## 阶段拆分

### 阶段 A：后端生成与中转底座

- 供应商适配层（统一请求/响应）
- 生成任务 API（创建、查询状态）
- 中转上传与存储回写（URL/file_id）
- 幂等缓存与失败重试
- 微信身份识别、周限额、积分扣减与失败回滚

### 阶段 B：小程序接入与降级

- 结果页改为消费后端统一图片引用
- 用户主动触发生图 + 授权弹窗
- 风格选择按钮（科技类/国潮类）
- 任务状态展示（生成中、已完成、失败）
- 失败自动回退本地静态图

### 阶段 C：数据与风控

- 生成图保留周期与清理策略
- 域名白名单与安全配置校验
- 供应商故障降级与开关控制
- 授权记录与配额/扣分审计可追踪

### 阶段 D：回归与验收

- 弱网与 5G 真机专项
- 供应商失败、超时、限流场景回归
- 页面与邮件一致性回归
- “拒绝授权即不生图”“同周超限拒绝”“积分不足拒绝”专项回归

## 非目标（本阶段不做）

- AI 图像风格玩法扩张（仅支持科技/国潮两个基础风格）
- 复杂积分运营体系（活动、商城、会员权益）不在本阶段
- 新增独立小程序或独立账号体系

## 验收门槛

- 真机（iOS/Android）体验版下，诗人图与国潮图可稳定展示
- 供应商临时不可用时，页面不空白，必须回退本地静态图
- 邮件与页面在同一请求内的图片来源可追踪
- 仅用户主动点击时才发起生图；未点击不触发
- 用户拒绝授权时不调用第三方生图
- 同一用户同一自然周第 2 次生图必须被拒绝并提示
- 每次成功生图均有积分扣减记录；失败回滚可验证
- 关键链路回归通过后才允许封板
