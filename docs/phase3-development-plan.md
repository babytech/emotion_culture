# 第三阶段开发计划（静态图池稳定链路）

## 文档目的

本文件用于明确第三阶段核心目标：
将诗人/国潮图片能力收敛为“预生成静态素材 + 云存储分发 + 稳定兜底”的可控链路，避免第三方动态图服务带来的成本与稳定性风险。

配套文档：

- [phase3-task-breakdown.md](/Users/babytech/github/emotion_culture/docs/phase3-task-breakdown.md)
- [phase3-implementation-checklist.md](/Users/babytech/github/emotion_culture/docs/phase3-implementation-checklist.md)

## 第三阶段目标（Must）

- 不再调用第三方动态生图服务
- 静态素材由运营预先生成并上传腾讯云存储
- 后端按情绪语境从图池中选择图片返回
- 小程序只加载后端返回的自有域名 URL 或 file_id
- 图片失败可回退本地兜底图，主分析结果不阻塞
- 页面与邮件图片来源口径一致可追踪
- 首页/结果页/分享页视觉进一步统一优化

## 固定方案

1. 运营提前准备科技/国潮素材并上传云存储
2. 后端维护素材元数据（字段固定为 `id,url,style,emotion_tags,intensity,active,weight,updated_at`）
3. 分析完成后按情绪语境筛选并加权随机选图
4. 同一用户近 7 天去重，降低重复感
5. 兜底顺序固定：风格候选池为空 -> 通用图池 -> 本地默认图
6. 小程序统一展示后端结果并给出明确状态提示
7. 建立每周素材运营机制：固定补图并按点击/收藏/邮件打开数据调整权重

## 元数据与兜底标准（Must）

- 元数据字段必须完整：`id,url,style(tech|guochao),emotion_tags,intensity,active,weight,updated_at`
- 图池选择必须遵循分层兜底：风格候选池 -> 通用图池 -> 本地默认图
- 当云存储 URL 失效时，按同一兜底链路回退，不影响主分析结果
- 运营侧每周至少一次补图与调权，且调权依据可追踪（点击/收藏/邮件打开）

## 阶段拆分

### 阶段 A：后端图池能力

- 元数据结构与加载逻辑（字段标准固定）
- 情绪语境匹配与加权随机
- 用户去重策略
- 云存储 URL 统一输出
- 回退兜底链路（候选池 -> 通用池 -> 本地默认）

### 阶段 B：小程序稳定展示

- 结果页图片来源统一
- 图片状态展示统一
- 弱网兜底体验优化
- 视觉与交互细节收口

### 阶段 C：运维与安全

- 素材上下线机制
- 每周补图与调权机制（点击/收藏/邮件打开）
- 域名白名单与访问策略校验
- 素材健康检查（失效 URL 识别）
- 页面与邮件一致性校验

### 阶段 D：回归与验收

- iOS/Android 真机回归
- 5G/弱网专项回归
- 页面与邮件一致性回归
- UI 视觉一致性与可读性回归

## 非目标（本阶段不做）

- 第三方动态生图服务接入（qwen/hunyuan/liblib 等）
- 动态生图积分扣费体系扩展
- 新账号体系与复杂会员体系

## 验收门槛

- 真机（iOS/Android）下科技/国潮图稳定展示
- 弱网下无长期空白图片区
- 页面与邮件图片来源一致
- 主分析结果在图片失败时仍可正常展示
- 三大页面视觉层级清晰，无重叠与遮挡
