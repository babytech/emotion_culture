# 第一阶段实施清单

## 文档目的

本文件用于将 [产品共识文档](/Users/babytech/github/emotion_culture/docs/product-consensus.md) 中已经达成一致的第一阶段范围，拆解为可执行的实施清单。

本文件只覆盖第一阶段“必须做”的能力，不包含第二阶段增强项。

## 实施原则

- 以稳定可用为优先目标，不追求第一阶段功能大而全
- 小程序与 `PC` 共用统一分析核心
- 语音先转文字，再进入统一分析链路
- 图像输入仅保留自拍或摄像头拍照，不保留普通上传图片分析
- 邮件为可选导出能力，不阻塞主结果展示
- 历史记录只保存轻量摘要，不长期保存原始媒体

## 范围总览

### 第一阶段交付物

- 小程序可稳定完成文字、语音、自拍三类输入分析
- `PC` 可稳定完成文字、录音、摄像头拍照三类输入分析
- 后端提供统一分析、邮件、历史记录相关能力
- 小程序提供结果页、简版历史记录页、极简设置页
- 历史记录、隐私边界、删除能力可用

### 第一阶段不纳入本清单

- 普通上传图片分析
- 实时视频流情绪识别
- 重账号体系
- 跨端统一身份与历史同步
- 情绪日历、趋势图、周报月报
- 积分体系、AI 画像奖励

## PC 实施清单

### 输入与界面

- [ ] 保留文字输入
- [ ] 保留录音输入
- [ ] 新增或完善摄像头拍照入口
- [ ] 拍照后支持预览与确认
- [ ] 去除“上传任意本地图片分析”主功能入口

### 摄像头拍照链路

- [ ] 拍照前检查摄像头可用性
- [ ] 拍照后做人脸可用性校验

### 历史与设置

- [ ] 第一阶段保留轻量本地历史能力
- [ ] `PC` 第一阶段不强制做完整历史页面
- [ ] `PC` 第一阶段可通过轻量菜单或弹窗承接必要设置项
