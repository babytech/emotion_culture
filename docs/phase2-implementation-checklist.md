## 实施原则

- 以留存闭环可用为优先目标，不追求第二阶段功能大而全
- 小程序与 `PC` 继续共用统一分析核心与留存聚合口径
- 端侧只做渲染和交互，聚合计算统一由后端输出
- 在不引入重会员体系前提下，优先做日历、打卡、周报、收藏
- 保持第一阶段隐私边界，原始媒体不进入长期留存
- `SPEECH_ASR_SERVICE` 是 ASR 开关，`VOICE_REQUIRE_TRANSCRIPT` 是严格度，不混用

## 范围总览

### 第二阶段交付物（方案 B）

- 小程序情绪日历页
- 小程序连续打卡能力
- 小程序周报回看能力

- `PC` 轻量留存回看（趋势/周报/收藏）
- 后端留存聚合 API 与数据策略

### 第二阶段不纳入本清单

- 多维检索与复杂筛选

- AI 海报/AI 头像/情绪明信片生成
- 跨端统一账号体系
