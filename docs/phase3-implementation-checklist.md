# 第三阶段实施清单（动态图稳定链路，必须完成）

## 文档目的

将第三阶段“动态图能力稳定化”目标拆解为可执行清单。  
本清单默认全部为必须项（Must），不再区分可选项。

配套文档：

- [phase3-development-plan.md](/Users/babytech/github/emotion_culture/docs/phase3-development-plan.md)
- [phase3-task-breakdown.md](/Users/babytech/github/emotion_culture/docs/phase3-task-breakdown.md)

## 后端实施清单（Must）

- [ ] 提供动态图任务创建接口
- [ ] 提供动态图任务状态查询接口
- [ ] 供应商适配层支持至少 1 种云服务与 1 种第三方服务扩展位
- [ ] 生成结果必须中转上传到自有存储
- [ ] 返回给小程序的图片引用必须为自有域名 URL 或 file_id
- [ ] 增加幂等缓存与失败重试
- [ ] 增加动态生图开关与故障降级
- [ ] 动态生图接口只允许微信身份用户（`openid/unionid`）访问
- [ ] 同一用户每自然周生图次数限制为 `1`
- [ ] 超过周限额时后端拒绝并返回明确错误码与提示文案
- [ ] 每次生图任务创建前完成积分扣减校验
- [ ] 生图任务失败时积分回滚可追踪

## 小程序实施清单（Must）

- [ ] 动态生图只能由用户主动点击触发（不自动触发）
- [ ] 点击生图前显示授权说明：将使用自拍原图并发送至第三方生图服务
- [ ] 用户拒绝授权时取消生图，不发起后端请求
- [ ] 提供风格选择按钮：`科技类`、`国潮类`
- [ ] 未选择风格不允许提交生图请求
- [ ] 结果页只消费后端统一图片引用
- [ ] 不直接依赖第三方图片域名
- [ ] 展示图片生成状态（生成中/失败/完成）
- [ ] 远端图片失败时自动回退本地静态图
- [ ] 兜底逻辑不影响主结果文本展示

## 数据与安全清单（Must）

- [ ] 明确生成图生命周期与清理策略
- [ ] 清理任务可回归验证
- [ ] 体验版/正式版域名策略一致
- [ ] 文档明确禁止端侧直连第三方动态图地址
- [ ] 记录用户授权决策（同意/拒绝）与时间戳
- [ ] 记录周限额命中、积分扣减、积分回滚审计字段
- [ ] 审计记录支持按 request_id / user_id 追踪

## 验收清单（Must）

- [ ] iOS 真机体验版：诗人/国潮图稳定展示
- [ ] Android 真机体验版：诗人/国潮图稳定展示
- [ ] 5G 弱网下可回退本地静态图，不出现长期空白
- [ ] 供应商超时/失败时，主分析结果不被阻塞
- [ ] 页面与邮件图片一致性可验证
- [ ] 用户拒绝授权后生图请求不会发出
- [ ] 同一用户同周第 2 次生图被拒绝并提示“本周仅可生成 1 次”
- [ ] 积分不足时生图被拒绝并提示补充积分
- [ ] 成功生图可看到积分扣减，失败生图可看到积分回滚

## 封板标准

仅当本清单全部勾选完成，第三阶段才可封板。
